# 系统服务异常 Runbook

## 常见现象

- 系统日志出现服务启动失败、崩溃、权限拒绝
- Docker 或 Kubernetes 日志出现容器退出、重启、健康检查失败
- 应用服务无法监听端口或依赖不可用

## 常见原因

- 服务配置错误
- 端口被占用
- 权限不足
- 依赖服务不可用
- 资源不足，例如 CPU、内存、文件句柄
- 发布或配置变更引入问题

## 关键日志关键词

- `failed`
- `permission denied`
- `address already in use`
- `timeout`
- `unavailable`
- `CrashLoopBackOff`
- `OOMKilled`

## 排查步骤

1. 确认日志来源、主机和故障时间点。
2. 查看同一时间段是否有相同服务的连续 ERROR/FATAL 日志。
3. 检查服务状态、端口监听和依赖服务状态。
4. 对比故障前是否有发布、配置变更或资源变化。
5. 如果是容器环境，检查容器退出码、重启次数和最近事件。

## 验证方法

- 确认服务状态恢复正常。
- 确认端口监听和健康检查通过。
- 观察同类错误日志是否停止增长。

## 风险提示

- 不要在证据不足时直接重启、删除或扩缩容生产服务。
- 修改配置前需要保留 diff 和回滚方案。
